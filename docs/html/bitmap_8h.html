Descripción detallada

Declaración de los métodos para representación de disco en el CRFS.

Fecha
28 May 2020 Se incluyen los métodos y representación para manejar el bloque bitmap de una partición del CRFS. Debido a la información que este tipo de bloque contiene, se debe mantener constantemente actualizado para representar correctamente el estado del disco. Por esto, se espera que estos métodos se ocupen tanto de forma interna por el sistema (para mantener la información actualizada) como externa (expuesto por la API para conocer el espacio libre).

void free_bitmap ( BitmapBlock *  bitmap)

Función para liberar un bloque de bitmap.

Esta función libera la memoria asociada a un bloque de bitmap. No se encarga de guardar esta información de ninguna forma, esto debe realizarse antes de liberar la memoria.

Parámetros
[in,out]bitmapEl bloque a liberar.

◆ init_bitmap()

BitmapBlock* init_bitmap ( unsigned char *  block,
int  uid 
)

Función para instanciar un bloque de bitmap.

Esta función recibe un array de integers, los cuales son interpretados como un un bloque de bitmap. Cada bit de esta esta array indica si cierto bloque esta disponible para escribir (0) o esta ocupado (1).

Si el array no es válido, se devuelve un puntero NULL.

Parámetros
[in]blockLa array de caracteres a interpretar.
[in]uidEl identificador del bloque a instanciar.
Devuelve
Un puntero al bloque instanciado.

◆ unload_bitmap()

unsigned char* unload_bitmap ( BitmapBlock *  bitmap)

Función para convertir un bloque bitmap a binario.

Esta función toma un bloque bitmap ya instanciado, y lo convierte a una representación binaria, tal como estaría en el archivo de disco virtual. Notese que la array vendra en Big Endian.

Parámetros
[in]bitmapEl bloque en cuestión.
Devuelve
La array de datos binarios.
